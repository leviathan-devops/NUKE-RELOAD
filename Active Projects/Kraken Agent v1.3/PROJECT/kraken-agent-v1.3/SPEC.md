# KRAKEN AGENT v1.3 — SPECIFICATION

## Version History
- **v1.0**: Initial ship - basic async orchestration with fake task execution
- **v1.1**: First multi-cluster attempt - throughput +983%
- **v1.2**: Triple-brain orchestrator 
- **v1.3**: Triple-brain + v2.0 narrative continuation (session lifecycle hooks)

## Purpose

Kraken v1.3 is a **self-contained orchestrator agent** with:
- **Triple-brain parallel architecture**: Planning / Execution / System brains run concurrently
- **Narrative continuation**: Session lifecycle hooks for compaction survival
- **Real Docker execution**: Direct spawn via opencode-python3 container
- **Two-Layer Guardian**: Tool enforcement + Message enforcement

## NARRATIVE CONTINUATION SYSTEM

### Session Lifecycle Hooks

| Hook | Trigger | Purpose |
|------|---------|---------|
| `event: session.created` | New session starts | Initialize all brains |
| `event: session.compacting` | Context being compressed | Write context to FILE for recovery |

### Compaction Flow

```
session.created
    │
    ▼
┌─────────────────────────────────────┐
│  1. Initialize Planning Brain        │
│  2. Initialize Execution Brain       │
│  3. Initialize System Brain          │
│  4. Load T2 Master context          │
│  5. Report: "All brains initialized"│
└─────────────────────────────────────┘
    │
    ▼ (when tokens running low)
session.compacting
    │
    ▼
┌─────────────────────────────────────┐
│  Write to /tmp/kraken-compaction/   │
│  LATEST/INJECTION.md                │
│                                     │
│  Contains:                          │
│  • Recent decisions                  │
│  • Current gate                     │
│  • Completed/active tasks           │
│  • Launchpad content               │
└─────────────────────────────────────┘
    │
    ▼ (session resumes)
session.created (recovery)
    │
    ▼
┌─────────────────────────────────────┐
│  Check for INJECTION.md            │
│  If exists → restore context        │
└─────────────────────────────────────┘
```

## BRAIN SPECIFICATIONS

### Planning Brain

**Owns:** `planning-state`, `context-bridge`

**Responsibilities:**
- Load T2 Master context from T2_*.md files
- Generate T1 from SPEC.md
- Decompose tasks for cluster assignment
- Domain ownership tracking

**Key Methods:**
```typescript
async loadT2Master(): Promise<void>
async generateT1(specPath: string): Promise<{ tasks: TaskSpec[], context: Record<string, unknown> }>
async decomposeTasks(t1: { tasks: TaskSpec[] }): Promise<TaskSpec[]>
designateCluster(taskType: string): 'alpha' | 'beta' | 'gamma'
```

### Execution Brain

**Owns:** `execution-state`, `quality-state`

**Responsibilities:**
- Direct task execution via subagent-manager
- Override authority (structured commands)
- Momentum tracking via OutputVerifier
- Subagent supervision

**Key Methods:**
```typescript
async executeTask(task: TaskSpec): Promise<TaskResult>
async superviseSubagent(taskId: string): Promise<void>
sendOverride(taskId: string, action: OverrideAction): void
verifyOutput(taskId: string, hostPath: string): boolean
```

### System Brain

**Owns:** `workflow-state`, `security-state`

**Responsibilities:**
- L0-L7 Firewall enforcement
- Two-Layer Guardian (tool + message)
- Gate evaluation with explicit criteria
- Evidence collection

**Key Methods:**
```typescript
checkTool(tool: string, args: Record<string, unknown>): void
checkMessage(message: string): void
evaluateGateEntry(gateId: string): GateResult
collectEvidence(gateId: string): Evidence[]
```

### Layer 1: Tool Enforcement (tool.execute.before)

Patterns checked:
- THEATRICAL_PATTERNS (counting theater)
- FAKE_TEST_PATTERNS
- WRONG_CONTAINER_PATTERNS

Patterns checked:
- THEATRICAL_MESSAGE_PATTERNS
- FAKE_COMPLETION_MESSAGE_PATTERNS
- HOST_FALLBACK_MESSAGE_PATTERNS
- SUCCESS_CLAIM_MESSAGE_PATTERNS

## OUTPUT VERIFICATION (Mechanical)

### V1.2: Trust-Based (WRONG)
```typescript
if (subagent.reportsSuccess()) markComplete();
```

### V1.3: Mechanical (CORRECT)
```typescript
interface OutputCheckpoint {
  containerPath: string;
  hostPath: string;
  verified: boolean;
}

// 1. Register outputs BEFORE task starts
outputVerifier.register(taskId, [
  { containerPath: '/app/dist/index.js', hostPath: '/host/dist/index.js' }
]);

// 2. After task completes, retrieve
await subagent.retrieveOutputs(paths);

// 3. MECHANICAL VERIFICATION
for (const path of paths) {
  if (!fs.existsSync(path)) {
    throw new Error(`L2_OUTPUT_NOT_RETRIEVED: ${path}`);
  }
}
```

## BRAIN COMMUNICATION

### Priority Messenger

```typescript
const PRIORITY = {
  critical: 0,  // Gate failures, security blocks
  high: 1,     // Context injections, state changes
  normal: 2,   // Checkpoints, routine updates
  low: 3,      // Logging, monitoring
};

// Example
messenger.send({
  from: 'kraken-system',
  to: 'kraken-execution',
  type: 'gate-failure',
  priority: 'critical',
  payload: { gateId: 'output-retrieval-gate', taskId: 'xyz' },
  requiresAck: true,
});
```

### Sync Points

Brains run in **parallel** and only synchronize at:

1. **task-assignment-gate** — Task assigned, cluster selected
2. **output-retrieval-gate** — Outputs retrieved, files verified
3. **roundtable-sync-gate** — All clusters reporting, no focus collision

## HOOK REGISTRATION (CRITICAL)

The hooks in `src/hooks/index.ts` are **DEFINED but NOT WIRED**.

In `src/index.ts`, the return object MUST register:

```typescript
return {
  name: 'kraken-agent',
  tool: allTools,
  config: ...,
  'event': createEventHook(),              // Session lifecycle
  'chat.message': createChatMessageHook(), // Layer 2 enforcement
  'tool.execute.before': createToolGuardianHook(), // Layer 1 enforcement
  'tool.execute.after': createGateHook(),  // Gate evaluation
};
```

## SUCCESS CRITERIA

1. **Session created → Brains initialize** — Console shows "All brains initialized"
2. **Tool call → Guardian fires** — Patterns checked on tool.execute.before
3. **Message → Guardian fires** — Patterns checked on chat.message
4. **Task complete → Output verified** — fs.existsSync() for declared outputs
5. **Compaction → Context preserved** — INJECTION.md written to /tmp/kraken-compaction/
6. **Docker container spawns** — opencode-python3:latest container runs agent
7. **Throughput ≥ 70 tasks/sec** — Parallel execution without await bottleneck
